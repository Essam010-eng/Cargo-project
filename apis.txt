API Endpoints (from existing code) — no code changes

Base URL: /api

====================
AUTH
====================

1) POST /api/auth/register
- Middleware: none
- Controller: middlewares/auth.register
- Body (req.body):
  - username (string)
  - password (string)
  - email (string)
  - role (string) => expected: "admin" or "seller"
- Gets from Data:
  - Checks existing user by email
  - If role == "admin", checks if an admin already exists
  - Creates new User in MongoDB
- Response: 201 JSON { user, token }

2) POST /api/auth/login
- Middleware: none
- Controller: middlewares/auth.login
- Body (req.body):
  - email (string)
  - password (string)
- Gets from Data:
  - Finds user by email
  - Compares password hash using bcrypt
- Response: 200 JSON { user, token }

Notes:
- Token payload: { id: user._id, role: user.role }, signed with process.env.secret_key, expiresIn: 1d

USERS
====================

Router mounted: app.js -> /api/user

3) GET /api/user/
- Middleware chain:
  - checkauth
  - checkrole("admin")
- Controller: controller/usercontroller.getAllUsers
- Gets from Data:
  - Finds all users in MongoDB, excluding password
- Response: 200 JSON { status:"success", results, users }

4) GET /api/user/:id
- Middleware chain:
  - checkauth
  - checkrole("admin")
- Controller: controller/usercontroller.getUserById
- Gets from Data:
  - Finds user by id in MongoDB, excluding password
- Response: 200 JSON { status:"success", user }

5) PUT /api/user/:id
- Middleware chain:
  - checkauth
- Controller: controller/usercontroller.updateMe
- Body (req.body):
  - username (optional)
  - email (optional)
- Gets from Data:
  - Updates currently authenticated user (req.user._id), NOT necessarily the :id param.
- Response: 200 JSON { status:"success", message, user }

6) GET /api/user/:id
- Middleware chain:
  - checkauth
- Controller: controller/usercontroller.getMe
- Gets from Data:
  - Finds user by req.user._id and returns it
- Response: 200 JSON { status:"success", user }

7) DELETE /api/user/:id
- Middleware chain:
  - checkauth
  - checkrole("admin")
- Controller: controller/usercontroller.deleteUser
- Gets from Data:
  - Deletes user by id in MongoDB
- Response: 200 JSON { status:"success", message:"User deleted successfully" }

Note:
- There is also a duplicated GET /api/user/:id route in routes/userrouter.js (one mapped to getMe, another exists mapped to getUserById). Order matters in Express.

  - checkrole("admin")
- Controller: controller/productcontroller.getOneProduct
- Gets from Data:
  - Product.findOne({ name })
- Response: 200 JSON (product)

10) POST /api/product/
- Middleware chain:
  - checkauth (via router.use)
  - upload.array("images", 5)
  - checkrole("admin","seller")
- Controller: controller/productcontroller.createProduct
- Body (req.body):
  - name
  - description
  - category
  - price
  - stock
  - carsforproduct (array of { name })
  - color (used inside colorimage object)
- File uploads:
  - req.files[].path is used to build colorimage.images
- Gets from Data:
  - Creates Product with:
    - colorimage: [{ color, images: [filePaths...] }]
    - owner: req.user.id
  - Computes average rating by aggregating Review for newProduct._id
  - Stores Product.averageRating
- Response: 201 JSON (created product)

11) PATCH /api/product/:id
- Middleware chain:
  - checkauth
  - upload.array("images", 5)
  - checkrole("admin")
- Controller: controller/productcontroller.updateproductpatch
- Body: (req.body fields passed directly to findByIdAndUpdate)
- Gets from Data:
  - Updates Product by id (code uses lowercase variable `product` but should be Product; depends on runtime)
- Response: res.send(updated product)

12) PUT /api/product/:id
- Middleware chain:
  - checkauth
  - upload.array("images", 5)
  - checkrole("admin")
- Controller: controller/productcontroller.updateproductpatch (same controller wired in router)
- Response: res.send(updated product)

13) DELETE /api/product/:id
- Middleware chain:
  - checkauth
  - checkrole("admin")
- Controller: controller/productcontroller.deleteproduct
- Gets from Data:
  - Deletes Product by id
- Response: res.send(deleted product)

RECOMMENDATIONS
====================

Router mounted: app.js -> /api (recomendationrouter is added as app.use("/api" , recomendationrouter))

29) GET /api/recommend/content?itemName=...
- Middleware: none
- Controller: controller/recomendationcontroller.getRecommendationByContent
- Query:
  - itemName (required)
- Gets from Data:
  - Calls external AI: {BASE_URL}/content-based with params { item_name: itemName }
  - Receives recommended product names
  - Product.find({ name: { $in: recommendedNames } })
- Response: 200 { success:true, data:[products] }

30) GET /api/recommend/trending?limit=10
- Middleware: none
- Controller: controller/recomendationcontroller.getTrendingRecommendations
- Query:
  - limit (optional, default 10)
- Gets from Data:
  - Calls external AI: {BASE_URL}/trending with params { top_n: limit }
  - Product.find({ name: { $in: trendingNames } }).lean()
  - For each product, Review.find({ product: product._id }) to compute liveAverageRating and totalReviewsCount
  - Sort by liveAverageRating desc
- Response: 200 { success:true, data:[products] }

31) GET /api/recommend/user?userId=...
- Middleware: none
- Controller: controller/recomendationcontroller.getCollaborativeRecommendations
- Query:
  - userId (required)
- Gets from Data:
  - Calls external AI: {BASE_URL}/collaborative with params { user_id: userId }
  - If empty -> fallback to AI trending (top_n=6) and returns those products
  - Else Product.find({ name: { $in: recommendedNames } })
- Response: 200 { success:true, data:[products] }
