============================================================
1) فكرة المشروع (Overview)
============================================================
مشروع “Car Go API” عبارة عن Backend REST API مبني بـ:
- Node.js + Express
- MongoDB + Mongoose
- JWT Authentication
- Upload للصور باستخدام Multer + Cloudinary (لمنتجات)
- رفع صورة وتحليل تلف/ضرر عبر AI خارجي (Card Damage)
- نظام توصيات عبر AI خارجي (Content/Trending/Collaborative) مع Fallback على قاعدة البيانات
- Swagger UI لتوثيق الـ API

5) التحقق من الهوية + الصلاحيات
============================================================
5.1) تحقق JWT: middlewares/checkauth.js
- يبحث عن token في الهيدر:
  - Authorization: Bearer <token>
- إذا لا يوجد token:
  - AppError(401, "You are not logged in, please log in to get access")
- verify token باستخدام:
  - process.env.secret_key
- ثم يبحث عن المستخدم في DB:
  - User.findById(decoded.id)
- إذا المستخدم غير موجود:
  - AppError(401, "The user belonging to this token no longer exists")
- عند النجاح:
  - req.user = finduser
  - next()
- أي خطأ في التحقق:
  - AppError(401, "Invalid token or token expired")

5.2) تحقق Role: middlewares/checkrole.js
- checkRole(...roles) يعيد middleware.
- إذا req.user غير موجود:
  - 401 Unauthorized
- إذا req.user.role غير موجود ضمن الأدوار المطلوبة:
  - 403 Forbidden
- غير ذلك next()

5.3) تسجيل/دخول: middlewares/auth.js
- register(req,res,next)
  - يقرأ:
    - username, password, email, address, role
  - يمنع email مكرر:
    - User.findOne({ email })
  - إذا role = admin:
    - يمنع وجود admin سابق (سجل admin واحد فقط)
  - bcrypt.hash(password, 10)
  - User.create
  - إنشاء JWT:
    - jwt.sign({ id: newUser._id, role: newUser.role }, secret_key, { expiresIn: "1d" })
  - يرجع 201:
    - { user: newUser, token }

- login(req,res,next)
  - يقرأ:
    - email, password
  - يجد المستخدم بالـ email
  - bcrypt.compare(password, user.password)
  - إنشاء JWT مماثل
  - يرجع 200:
    - { user, token }

7) النماذج (Mongoose Models)
============================================================
7.1) Product: models/products.js
حقول أساسية:
- name, description, category: String required
- price: Number required
- stock: Number required
- colorimage: array من objects:
  - color
  - images: [String]
- carsforproduct: array من objects:
  - name (car model)
- owner: ObjectId required
- averageRating: Number default 0
- createdAt: Date default now

============================================================
7.2) User: models/user.js
- username: String required + trim
- password: String required + minlength 6
- email: String required + unique + lowercase
- address: String (مطلوب)
- role: enum ["admin","seller","user"] default "user"
- createdAt: Date default now

============================================================
7.3) Review: models/reviwe.js
- user: ObjectId ref 'User' required
- product: ObjectId ref 'Product' required
- rating: Number min 1 max 5 required
- comment: String required
- timestamps: true

حساب متوسط التقييم:
- يوجد static calcAverageRatings(productId)
  - aggregate:
    - match: product = productId
    - group:
      - avgRating: avg(rating)
      - ratingsQuantity: sum(1)
  - ثم update على Product:
    - ratingsAverage / ratingsQuantity

ملاحظة مهمة داخل الكود:
- Product model يحوي averageRating
- بينما calcAverageRatings يقوم بتحديث ratingsAverage/ratingQuantity
(هذا اختلاف حقول داخل الكود كما هو).

Hook:
- post('save') -> this.constructor.calcAverageRatings(this.product)

============================================================
7.4) Order: models/order.js
- user: ObjectId ref "User" required
- products: array من عناصر:
  - product: ObjectId ref "Product" required
  - quantity: Number min 1 required
  - price: Number required
- totalPrice: Number (يتم حسابها)
- status: enum ['pending','shipping','delivered'] default 'pending'
- createdAt: Date default now

Hook pre('save'):
- totalPrice = sum(quantity * price)

8) Controllers (المنطق داخل الـ API)
============================================================
8.1) Product Controller: controller/productcontroller.js
- getAllProducts
  - Product.find()
  - إذا لا يوجد منتجات: 404 "No products found"

- getAllProductsforsearche
  - بناء query فارغ
  - إذا req.query.name موجود -> regex case-insensitive
  - Product.find(query)

- getOneProduct
  - يستقبل req.params.id
  - Product.findOne({ id })
  - 404 إذا لا يوجد

- createProduct
  - يقرأ fields من req.body
  - filePaths = req.files.map(file => file.path)
  - Product.create مع:
    - colorimage: [{ color, images: filePaths }]
    - owner: req.user.id

- updateproductpatch
  - findByIdAndUpdate(req.params.id, req.body, { new: true })

- updateproductput
  - findByIdAndUpdate(req.params.id, req.body, { new: true, overwrite: true })

- deleteproduct
  - findByIdAndDelete(req.params.id)

Seller-specific:
- sellergetallproduct
  - Product.find({ owner: req.user._id })

- sellergetoneproduct
  - findOne({_id: id, owner: req.user.id})

- sellerupdatepatch
  - findByIdAndUpdate({ _id, owner }, req.body, { new: true })

- sellerupdateput
  - findByIdAndUpdate({ _id, owner }, req.body, { new: true, overwrite: true })

- sellerdeleteproduct
  - findByIdAndDelete({ _id, owner })

8.5) Recommendation Controller: controller/recomendationcontroller.js
يعتمد على AI خارجي عبر axios.

- BASE_URL = 'https://abdoelhadray-recommendation-system.hf.space/recommend'

1) getRecommendationByContent
- Query param: itemName
- إذا itemName غير موجود -> 400 رسالة واضحة
- AI call:
  - GET `${BASE_URL}/content-based` params: { item_name: itemName }
  - timeout 3000
- إذا AI failed -> fallback:
  - Product.find({}).limit(10)
- إذا رجعت recommendedNames:
  - Product.find({ name: { $in: recommendedNames } }).lean()
  - sort حسب ترتيب recommendedNames

2) getTrendingRecommendations
- Query param: limit (default 10)
- AI call:
  - GET `${BASE_URL}/trending` params: { top_n: limit }
  - timeout 3000
- معالجة response قد تكون objects -> استخراج Name/name
- إذا المنتجات رجعت empty -> fallback:
  - Product.find({}).sort({ averageRating: -1 }).limit(limit)
- ثم حساب “live” متوسط rating من Review:
  - تجمع كل reviews للـ products
  - لكل Product:
    - totalRating = sum(r.rating)
    - liveAverageRating = totalRating/reviews.length
    - totalReviewsCount = reviews.length
  - sort نهائي حسب liveAverageRating desc

3) getCollaborativeRecommendations
- Query param: userId
- AI call:
  - GET `${BASE_URL}/collaborative` params: { user_id: userId }
  - timeout 3000
- إذا AI failed أو لا يوجد تطابق -> fallback:
  - Product.find({}).sort({ averageRating: -1 }).limit(6)

12) ملاحظات تقنية/سلوكية داخل الكود (بدون تعديل)
============================================================
- Review.calcAverageRatings يقوم بتحديث:
  - ratingsAverage و ratingsQuantity
بينما Product model يحتوي:
  - averageRating
(هذا اختلاف أسماء داخل الكود قد يؤثر على ظهور متوسط التقييم).

- productcontroller.getOneProduct يستخدم:
  - Product.findOne({ id })
بينما بعض العمليات الأخرى تستخدم _id.
(قد يسبب عدم تطابق حسب شكل البيانات القادمة من العميل).

- في productrouter:
  - PUT /api/product/:id يستدعي updateproductpatch بدل updateproductput.

- يوجد router.use(checkauth) داخل productRouter، ما يعني حماية معظم مسارات المنتجات بعده.

13) الخلاصة
============================================================
المشروع يقدم backend REST API لسيستم:
- Auth JWT (register/login)
- إدارة المنتجات (CRUD + Upload صور للـ Cloudinary)
- إدارة المراجعات (Reviews) وربطها بالـ users و products
- إدارة الطلبات (Orders) وحساب totalPrice مع تغيير status
- كشف تلف/ضرر في صور عبر AI خارجي
- نظام توصيات ثلاثي (Content/Trending/Collaborative) عبر AI خارجي + fallback على MongoDB
- Swagger UI لتوثيق وتصفح الـ API
